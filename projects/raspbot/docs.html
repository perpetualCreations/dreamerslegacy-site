1.1: Introduction

Raspbot is a small robotic vehicle, that is remotely operated via a local network, through a custom client application.

The onboard computer is powered by a Raspberry Pi 3 B+ running Raspbian. Additionally, it contains an Arduino microcontroller and an Arduino Motor Shield Rev3 (for controlling the dual tank treads).

1.3: Remote Control Application (RCA)

As mentioned, the robotic vehicle is operated via a client and host application. The client is GUI-based, the previous client was based on a command-line interface, the next releases of the control application will possibly make a return to CLI, for low-resource or specific scenarios where CLI is preferred. The host Raspbot RCA runs on Debian or Debian-based systems (i.e Ubuntu, Raspbian). The client is not OS-dependent.

The application is written in Python, you may run it by installing Python 3 and the application's associated packages with PIP, please see requirements.txt files for package and version requirements. If using the SenseHAT as part of your hardware configuration, please install the PyPI package and also the Debian package. See hardware list for more details on SenseHAT and certain packages for particular hardware.

Client binaries will be made available, however they are unsigned.

It's recommended to use a designated user account on the Debian or Debian-based system, and to extract release contents to the user's home directory.

In Raspbot RCA v1.1, credits to Pyimagesearch for the live camera view, please see /tkinter-photo-booth/tkinter-photo-booth-citation.txt for more information. Please note v1.2 and above this dependency has been removed.

1.4: Previous Revision

The previous hardware revision used a single-use 9V battery for supplying motor power, and used a commercial-off-the-shelf battery bank to power the Raspberry Pi. The chassis has since been redesigned, with more mounting space and wire pass-throughs.

The control application client was updated from CLI to use Tkinter, and now the Qt GUI framework through PySide6. Since v1.1, the feature set and scope has gradually grown, and recently the Arduino code was updated for multi-character commands using a buffer.
