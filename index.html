Personal Package Archive (PPA)

Part of this portfolio site hosts the Personal Package Archive (PPA) for the distribution of my Debian packages. You can add it to your APT sources list to install its offerings.

From testing, I can inform you this will likely work on most Debian and Debian-based operating systems, including Raspbian, however expect broken dependencies for certain packages.

To add the archive repository, run:

sudo curl -s --compressed "https://dreamerslegacy.xyz/ppa/ubuntu/KEY.gpg" | sudo apt-key add -

sudo curl -s --compressed -o /etc/apt/sources.list.d/dreamerslegacy_ppa.list "https://dreamerslegacy.xyz/ppa/ubuntu/dreamerslegacy_ppa.list"

sudo apt update

To remove the archive repository, run:

sudo apt-key del 9BBEC5D61BBD56C2CCC437D2A4891A46DC4C3713

sudo rm /etc/apt/sources.list.d/dreamerslegacy_ppa.list

sudo apt update

This method adds/removes a sources file under /etc/apt/sources.list.d/ and the repository public GPG key for package signing.

If typing out these commands is burdensome, see this Github release for two shell scripts that perform the above commands (perhaps a bit pointless, considering the trouble you need to go through, compared to copy-and-pasting commands).

deargodpleaseno

Ever wanted to host a webpage or file for a few days or hours, before it is deleted and wiped off the internet?

deargodpleaseno is the manifestation of the obsessive will of the Python programmer to automate everything.

Available as a Debian package, you can download and install deargodpleaseno on your Apache web server.

Upon installation, a new folder will be created in the webroot, literally called "deargodpleaseno" however is configurable. Temporary files go here.

Over CLI, you can request a list of temporary files, set the time and date of expiration, and see who has requested the resource.

Additionally, deargodpleaseno disallows access of the temporary files through user-agent rules. Anything inside the directory (assuming the crawler follows user-agent rules), should not be indexed by search engines.

It also allows for remote user control over FTP, SSH/SFTP, and Telnet (however these are not built in, they merely are automatically installed when not present and at the user's request. Raspbot RCA-G is a GUI-based bot controller written in both Python and partially in Arduino C. It features a live camera feed, SenseHAT integration for LED display and sensor inputs, navigational input over serial to an Arduino Motor Shield Rev3, and finally more sensor inputs from Grove sensors*.

These will be released hopefully soon. My reputation depends on it after all.

*currently in-dev, not hardware verified yet.
